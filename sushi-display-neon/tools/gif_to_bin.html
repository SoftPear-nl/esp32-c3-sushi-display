GIF → packed RGB565 .bin for ESP32 / ST7789

Decodes every frame of an animated GIF and packs them into a single raw binary file.
Each frame is width × height × 2 bytes of big-endian RGB565 (byte-swapped, ready for ST7789 over SPI).
A tiny 8-byte header is prepended: W u16 H u16 frames u16 fps u16 — all little-endian.
No ZIP, no BMP headers — fseek(f, 8 + frame*W*H*2, SEEK_SET) and fread is all it takes on the ESP32.

ESP32 C usage example 
// After download, place the .bin in your flash_fs/ folder
// and flash with SPIFFS.

#pragma pack(push,1)
typedef struct { uint16_t w, h, frames, fps; } bin_hdr_t;
#pragma pack(pop)

void play_bin(esp_lcd_panel_handle_t panel, const char *path)
{
    FILE *f = fopen(path, "rb");
    bin_hdr_t hdr;
    fread(&hdr, sizeof(hdr), 1, f);

    size_t frame_bytes = (size_t)hdr.w * hdr.h * 2;
    uint16_t *buf = heap_caps_malloc(frame_bytes, MALLOC_CAP_SPIRAM);

    uint32_t frame_ms = hdr.fps ? (1000 / hdr.fps) : 100;

    for (;;) {
        fseek(f, sizeof(hdr), SEEK_SET);
        for (int i = 0; i < hdr.frames; i++) {
            fread(buf, 2, hdr.w * hdr.h, f);
            esp_lcd_panel_draw_bitmap(panel, 0, 0, hdr.w, hdr.h, buf);
            vTaskDelay(pdMS_TO_TICKS(frame_ms));
        }
    }
}
